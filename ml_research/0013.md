# 72. Comparing micrograd to PyTorch

After building micrograd’s scalar autograd engine, Karpathy shows that the same computation can be done in PyTorch.

The purpose is not to teach PyTorch deeply yet.

The purpose is to show:

```text id="3lqq7l"
micrograd and PyTorch are conceptually doing the same thing
```

Micrograd has:

```python id="fak18e"
Value
```

PyTorch has:

```python id="bl3df4"
torch.Tensor
```

Micrograd stores:

```text id="df17dy"
.data
.grad
.backward()
```

PyTorch tensors also have:

```text id="th33vp"
.data
.grad
.backward()
```

So micrograd has been designed to mimic the conceptual surface of PyTorch.

---

# 73. Scalar `Value` vs tensor

Micrograd works with scalar values:

```python id="1q17l3"
a = Value(2.0)
```

PyTorch works with tensors:

```python id="i20cwf"
a = torch.Tensor([2.0])
```

A tensor can be:

```text id="cirvim"
a scalar-like one-element array
a vector
a matrix
a higher-dimensional array
```

Example tensor:

```python id="x61mfp"
torch.Tensor([[1, 2, 3], [4, 5, 6]])
```

has shape:

```text id="d8sz6q"
2 × 3
```

The tensor abstraction is what makes PyTorch efficient.

Instead of tracking and operating on one scalar at a time, PyTorch tracks arrays of scalars and runs operations on many values in parallel.

But mathematically:

```text id="z9d835"
same derivatives
same chain rule
same backpropagation idea
```

---

# 74. Why PyTorch examples use `.double()`

Karpathy uses one-element tensors to match micrograd’s scalar behavior.

Example:

```python id="j93xmw"
x1 = torch.Tensor([2.0]).double()
```

By default, PyTorch often uses `float32`.

Python floats are usually double precision, equivalent to `float64`.

So he calls:

```python id="2950zg"
.double()
```

to make PyTorch use double precision and match micrograd’s numerical values more closely.

Purpose:

```text id="tlf1ly"
avoid tiny numerical discrepancies due to float precision
```

---

# 75. Why PyTorch examples use `requires_grad=True`

In PyTorch, tensors do not automatically track gradients unless asked.

So Karpathy does:

```python id="7l7l1j"
x1.requires_grad = True
x2.requires_grad = True
w1.requires_grad = True
w2.requires_grad = True
b.requires_grad = True
```

or equivalently in modern style:

```python id="aos5jy"
x1 = torch.tensor([2.0], dtype=torch.float64, requires_grad=True)
```

Reason:

```text id="h941qi"
gradient tracking has overhead
```

In real neural network training, you usually want gradients for parameters, not for all temporary or input tensors.

PyTorch defaults are efficiency-conscious.

# 76. Recreating the neuron in PyTorch

The neuron expression is the same one used earlier:

```text id="4ve1wm"
o = tanh(x1*w1 + x2*w2 + b)
```

With:

```text id="drru9x"
x1 = 2
x2 = 0
w1 = -3
w2 = 1
b = 6.881373587...
```

PyTorch version:

```python id="tuwdds"
n = x1*w1 + x2*w2 + b
o = torch.tanh(n)
```

Then:

```python id="d4ctdj"
o.backward()
```

PyTorch computes gradients.

Expected results:

```text id="ddj5i9"
o.data ≈ 0.7071

x1.grad ≈ -1.5
w1.grad ≈ 1.0
x2.grad ≈ 0.5
w2.grad ≈ 0.0
b.grad  ≈ 0.5
```

These match micrograd.

This validates the micrograd implementation against the behavior of a real production autograd library.

# 80. Neural networks as a specific class of expressions

The autograd engine does not know about neural networks.

It only knows:

```text id="8hboi6"
Value objects
operations
graphs
gradients
```

The neural network library simply uses those tools to build a particular kind of mathematical expression.

For a neural network:

```text id="o6duya"
inputs and parameters go in
a prediction comes out
```

Because the prediction is made of `Value` operations, micrograd can backpropagate through it.

---

# 81. Implementing a `Neuron`

A neuron needs:

```text id="8iwehg"
weights
bias
forward call
```

Constructor:

```python id="2trn5y"
class Neuron:
    def __init__(self, nin):
        self.w = [Value(random.uniform(-1, 1)) for _ in range(nin)]
        self.b = Value(random.uniform(-1, 1))
```

Here:

```text id="97b81m"
nin = number of inputs to the neuron
```

If `nin = 3`, the neuron has:

```text id="qltrvm"
3 weights
1 bias
```

Each weight is randomly initialized between `-1` and `1`.

---

# 82. Why random initialization?

If all neurons started with identical weights, they would compute identical functions and receive identical gradients.

Random initialization breaks symmetry.

Here the initialization is simple and pedagogical:

```python id="v85mc2"
random.uniform(-1, 1)
```

No advanced initialization scheme is used because the network is tiny.

---

# 83. `__call__`: making a neuron callable

Karpathy implements:

```python id="lcnyrc"
def __call__(self, x):
```

This lets a neuron object be used like a function:

```python id="54y3n3"
n = Neuron(2)
out = n(x)
```

Python internally translates:

```python id="4mgia6"
n(x)
```

into:

```python id="zmn275"
n.__call__(x)
```

This mirrors PyTorch modules, which are also called like functions.

---

# 84. Neuron forward pass

The neuron computes:

```text id="lagz1a"
activation = w · x + b
output = tanh(activation)
```

Expanded:

```text id="linv6n"
activation = w1*x1 + w2*x2 + ... + wn*xn + b
```

Implementation:

```python id="qvp354"
def __call__(self, x):
    act = sum((wi*xi for wi, xi in zip(self.w, x)), self.b)
    out = act.tanh()
    return out
```

Important pieces:

```text id="0aeql5"
zip(self.w, x) pairs each weight with the corresponding input
wi*xi multiplies each pair
sum(..., self.b) adds all products, starting from the bias
act.tanh() applies the nonlinearity
```

The result is a `Value`.

Because all operations are `Value` operations, a computation graph is built automatically.

---

# 85. Understanding `zip(self.w, x)`

If:

```python id="1rtozq"
self.w = [w1, w2]
x = [x1, x2]
```

then:

```python id="7hq5yd"
zip(self.w, x)
```

iterates over:

```text id="lt925s"
(w1, x1)
(w2, x2)
```

So:

```python id="ckm6qb"
(wi*xi for wi, xi in zip(self.w, x))
```

generates:

```text id="51kog8"
w1*x1
w2*x2
```

Then `sum` adds them.

---

# 86. Why `sum(..., self.b)` starts from the bias

Python’s `sum` can take a second argument:

```python id="gma7hj"
sum(iterable, start)
```

The default `start` is `0`.

Karpathy uses:

```python id="gtw705"
sum((wi*xi for wi, xi in zip(self.w, x)), self.b)
```

so the sum starts from the bias.

This gives:

```text id="deavlv"
b + w1*x1 + w2*x2 + ...
```

instead of:

```text id="hqvf7f"
0 + w1*x1 + w2*x2 + ...
```

It is concise and directly expresses the neuron equation.

# 88. Implementing a `Layer`

A layer is a collection of neurons evaluated independently on the same input.

Constructor:

```python id="v02oss"
class Layer:
    def __init__(self, nin, nout):
        self.neurons = [Neuron(nin) for _ in range(nout)]
```

Here:

```text id="m4vhgz"
nin = number of inputs to each neuron
nout = number of neurons in the layer
```

If:

```python id="wu4cda"
Layer(2, 3)
```

then:

```text id="h84lu4"
3 neurons
each neuron has 2 inputs
```

---

# 89. Layer forward pass

Implementation:

```python id="3fgkjx"
def __call__(self, x):
    outs = [n(x) for n in self.neurons]
    return outs[0] if len(outs) == 1 else outs
```

Each neuron receives the same input vector `x`.

If the layer has multiple neurons, it returns a list of outputs.

If the layer has exactly one neuron, it returns the single output directly instead of wrapping it in a one-item list.

This convenience matters for the final output layer.

---

# 90. Testing a layer

Example:

```python id="ad07xr"
x = [2.0, 3.0]
layer = Layer(2, 3)
layer(x)
```

Returns something like:

```text id="oev46d"
[Value(...), Value(...), Value(...)]
```

Each output is the result of an independent neuron.

Conceptually:

```text id="2d3ppw"
same input
→ multiple neurons
→ multiple outputs
```

# 91. Implementing an MLP

An MLP is a sequence of layers.

Constructor:

```python id="osdjfe"
class MLP:
    def __init__(self, nin, nouts):
        sz = [nin] + nouts
        self.layers = [Layer(sz[i], sz[i+1]) for i in range(len(nouts))]
```

Here:

```text id="lvut0f"
nin = input dimensionality
nouts = list of layer sizes
```

Example:

```python id="n3ueaz"
MLP(3, [4, 4, 1])
```

means:

```text id="w3qb27"
input size: 3
first hidden layer: 4 neurons
second hidden layer: 4 neurons
output layer: 1 neuron
```

The list:

```python id="7s9d3y"
sz = [3] + [4, 4, 1]
```

becomes:

```python id="7bqrer"
sz = [3, 4, 4, 1]
```

Then consecutive pairs define layers:

```text id="kze8su"
Layer(3, 4)
Layer(4, 4)
Layer(4, 1)
```

---

# 92. MLP forward pass

Implementation:

```python id="qz8ptw"
def __call__(self, x):
    for layer in self.layers:
        x = layer(x)
    return x
```

This feeds the output of each layer into the next layer.

Flow:

```text id="3w8dhy"
input x
→ layer 1
→ layer 2
→ output layer
→ prediction
```

Because every neuron output is a `Value`, the whole MLP produces a computation graph.

Network:


```text id="9b84gr"

```

Karpathy visualizes:


The graph is much larger than the earlier toy examples.

It contains:

