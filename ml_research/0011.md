# 18. Manual backpropagation example #1: simple expression

We continue from the expression:

```python
a = Value(2.0, label='a')
b = Value(-3.0, label='b')
c = Value(10.0, label='c')

e = a * b
e.label = 'e'

d = e + c
d.label = 'd'

f = Value(-2.0, label='f')

L = d * f
L.label = 'L'
```

Forward values:

```text
a = 2
b = -3
c = 10
e = a*b = -6
d = e+c = 4
f = -2
L = d*f = -8
```

Graph:

```text
a ----\
      * → e ----\
b ----/          + → d ----\
c ---------------/          * → L
f --------------------------/
```

Backpropagation means filling in:

```text
L.grad
d.grad
f.grad
e.grad
c.grad
a.grad
b.grad
```

where each `.grad` stores:

```text
∂L/∂that_node
```

---

# 19. Base case: derivative of output with respect to itself

The first derivative is:

```text
∂L/∂L = 1
```

Reason:

```text
if L changes by h, L changes by h
```

So:

```python
L.grad = 1.0
```

This is always the starting point of backpropagation.

In a neural network, the final output is usually the loss, so this means:

```text
∂loss/∂loss = 1
```

---

# 20. Backprop through multiplication: `L = d * f`

We have:

```text
L = d * f
```

We need:

```text
∂L/∂d
∂L/∂f
```

For multiplication:

```text
if L = d*f
then ∂L/∂d = f
and  ∂L/∂f = d
```

Using the current forward values:

```text
d = 4
f = -2
```

Therefore:

```text
∂L/∂d = f = -2
∂L/∂f = d = 4
```

So:

```python
d.grad = -2.0
f.grad = 4.0
```

Interpretation:

```text
increasing d slightly decreases L, because f is negative
increasing f slightly increases L, because d is positive
```

Numerical check:

```python
h = 0.0001
```

If you nudge `f`:

```text
f += h
L changes by about 4h
```

so:

```text
∂L/∂f ≈ 4
```

If you nudge `d`:

```text
d += h
L changes by about -2h
```

so:

```text
∂L/∂d ≈ -2
```

---

# 21. The key backprop idea: local derivatives + chain rule

Now we want to go further backward from `d` to `e` and `c`.

We know:

```text
d = e + c
```

and we already know:

```text
∂L/∂d = -2
```

But we need:

```text
∂L/∂e
∂L/∂c
```

This is the core of backpropagation.

The local derivatives are:

```text
∂d/∂e = 1
∂d/∂c = 1
```

because:

```text
d = e + c
```

A plus operation passes changes straight through.

But the quantity we need is not merely:

```text
∂d/∂e
```

We need:

```text
∂L/∂e
```

That requires the chain rule.

---

# 22. Chain rule

General form:

```text
if z depends on y
and y depends on x
then z depends on x
```

The chain rule says:

```text
dz/dx = dz/dy * dy/dx
```

For this graph:

```text
L depends on d
d depends on c
therefore L depends on c
```

So:

```text
∂L/∂c = ∂L/∂d * ∂d/∂c
```

We know:

```text
∂L/∂d = -2
∂d/∂c = 1
```

Therefore:

```text
∂L/∂c = -2 * 1 = -2
```

Similarly:

```text
∂L/∂e = ∂L/∂d * ∂d/∂e
       = -2 * 1
       = -2
```

So:

```python
c.grad = -2.0
e.grad = -2.0
```

---

# 23. Plus nodes distribute gradients

For:

```text
d = e + c
```

the plus node’s local derivative with respect to each input is `1`.

Therefore, during backprop:

```text
incoming gradient to d = -2
```

gets copied to both children:

```text
e.grad = -2
c.grad = -2
```

So a useful rule:

```text
plus node:
routes incoming gradient unchanged to all inputs
```

More generally:

```python
x = a + b
```

during backprop:

```text
a.grad += x.grad
b.grad += x.grad
```

The `+=` part becomes important later.

---

# 24. Backprop through multiplication: `e = a * b`

Now we know:

```text
∂L/∂e = -2
```

and:

```text
e = a * b
```

We need:

```text
∂L/∂a
∂L/∂b
```

Use chain rule:

```text
∂L/∂a = ∂L/∂e * ∂e/∂a
∂L/∂b = ∂L/∂e * ∂e/∂b
```

For multiplication:

```text
e = a*b
∂e/∂a = b
∂e/∂b = a
```

Forward values:

```text
a = 2
b = -3
```

Therefore:

```text
∂L/∂a = (-2) * (-3) = 6
∂L/∂b = (-2) * (2) = -4
```

So:

```python
a.grad = 6.0
b.grad = -4.0
```

Full gradients:

```text
L.grad = 1
d.grad = -2
f.grad = 4
e.grad = -2
c.grad = -2
a.grad = 6
b.grad = -4
```

# 25. What backprop is doing conceptually

Backpropagation is not magic.

It repeatedly applies this recipe:

```text
1. Know the gradient of the final output with respect to the current node.
2. Look at the operation that produced the current node.
3. Compute local derivatives of that operation.
4. Multiply by the incoming gradient using chain rule.
5. Pass the result backward to the operation’s inputs.
```

For a node:

```text
out = operation(inputs)
```

you know:

```text
∂L/∂out
```

and compute:

```text
∂L/∂input_i = ∂L/∂out * ∂out/∂input_i
```

This is all backpropagation is:

```text
recursive chain rule through a computation graph
```

# 26. Preview of optimization: nudging inputs to increase `L`

Now Karpathy shows what the gradients can be used for.

Current value:

```text
L = -8
```

Suppose we want to make `L` increase.

The gradient tells us how each leaf affects `L`:

```text
a.grad = 6
b.grad = -4
c.grad = -2
f.grad = 4
```

To increase `L`, move each leaf slightly in the direction of its gradient:

```python
step_size = 0.01

a.data += step_size * a.grad
b.data += step_size * b.grad
c.data += step_size * c.grad
f.data += step_size * f.grad
```

This gives:

```text
a increases, because a.grad is positive
b decreases, because b.grad is negative
c decreases, because c.grad is negative
f increases, because f.grad is positive
```

Then recompute the forward pass:

```python
e = a * b
d = e + c
L = d * f
```

The new `L` becomes less negative:

```text
L increases from about -8 to about -7.286
```

This is one step of optimization.

The pattern is:

```text
compute output
compute gradients
change inputs/parameters using gradients
recompute output
```

For neural network training, the goal will usually be the opposite:

```text
make loss smaller
```

So later we will move parameters in the **negative** gradient direction.

# 27. Manual backpropagation example #2: a neuron

Now Karpathy moves from toy scalar expressions toward neural networks.

A simple neuron has:

```text
inputs x
weights w
bias b
activation function
```

For a two-input neuron:

```text
x1, x2 = inputs
w1, w2 = weights
b = bias
```

The neuron computes:

```text
n = x1*w1 + x2*w2 + b
o = tanh(n)
```

Where:

```text
n = raw activation
o = output
```

The weights determine how strongly each input affects the neuron.

The bias controls the neuron’s trigger-happiness independent of the inputs.

The activation function squashes the raw value into a bounded range.

---

# 28. The `tanh` activation

Karpathy uses `tanh`.

Shape:

```text
large negative input → output near -1
zero input           → output 0
large positive input → output near +1
```

So:

```text
tanh squashes real numbers into [-1, 1]
```

This introduces nonlinearity.

Without nonlinearities, stacking layers would collapse into one linear operation. Nonlinearity is what allows neural networks to represent more complex functions.

---

# 29. Building the neuron expression

Karpathy creates specific scalar values:

```python
x1 = Value(2.0, label='x1')
x2 = Value(0.0, label='x2')

w1 = Value(-3.0, label='w1')
w2 = Value(1.0, label='w2')

b = Value(6.8813735870195432, label='b')
```

The bias is chosen carefully so the numbers during backprop come out nicely.

Intermediate products:

```python
x1w1 = x1 * w1
x1w1.label = 'x1*w1'

x2w2 = x2 * w2
x2w2.label = 'x2*w2'
```

Then:

```python
x1w1x2w2 = x1w1 + x2w2
x1w1x2w2.label = 'x1*w1 + x2*w2'

n = x1w1x2w2 + b
n.label = 'n'
```

Forward values:

```text
x1*w1 = 2 * -3 = -6
x2*w2 = 0 * 1 = 0
x1*w1 + x2*w2 = -6
n = -6 + 6.881373587... = 0.881373587...
```

Then:

```python
o = n.tanh()
o.label = 'o'
```

Output:

```text
o = tanh(n) ≈ 0.7071
```

# 30. Why implement `tanh` directly?

At this point, the `Value` class only supports:

```text
+
*
```

But `tanh` mathematically involves exponentials:

```text
tanh(x) = (e^(2x) - 1) / (e^(2x) + 1)
```

Karpathy says there are two possible approaches:

```text
1. implement exp, subtraction, division, etc., then build tanh from them
2. implement tanh directly as a single operation
```

He chooses option 2 first.

Important design principle:

```text
autograd operations do not have to be atomic
```

An operation can be simple:

```text
+
*
```

or composite:

```text
tanh
```

The only requirement:

```text
for every operation, we must know how to compute its local derivative
```

So `tanh` can be a single node in the graph if we know its derivative.

# 31. Adding `tanh` to `Value`

The method looks conceptually like:

```python
def tanh(self):
    x = self.data
    t = (math.exp(2*x) - 1) / (math.exp(2*x) + 1)
    out = Value(t, (self,), 'tanh')
    return out
```

Important details:

```text
input: one Value object
output: new Value object
child pointer: points back to the input
operation label: 'tanh'
```

So:

```python
o = n.tanh()
```

creates a graph edge:

```text
n → tanh → o
```

Forward value:

```text
n ≈ 0.8814
o ≈ 0.7071
```

```text
∂o/∂all previous nodes
```

```text
∂o/∂o = 1
```

So:

```python
o.grad = 1.0
```

# 33. Backprop through `tanh`

We have:

```text
o = tanh(n)
```

Derivative:

```text
d/dn tanh(n) = 1 - tanh(n)^2
```

But:

```text
o = tanh(n)
```

So:

```text
∂o/∂n = 1 - o²
```

Since:

```text
o ≈ 0.7071
```

then:

```text
o² ≈ 0.5
1 - o² ≈ 0.5
```

So:

```python
n.grad = 0.5
```

Interpretation:

```text
a tiny increase in n increases o with slope about 0.5
```

Because `tanh` is nonlinear, its derivative depends on where you are.

Near zero, tanh has high slope.

Near ±1, tanh saturates and the slope becomes small.

Again addition.

```text
x1w1x2w2.grad = 0.5
```

So:

```python
x1w1.grad = 0.5
```

# 36. Backprop through `x2w2 = x2 * w2`

We have:

```text
x2w2 = x2 * w2
```

Incoming gradient:

```text
x2w2.grad = 0.5
```

Multiplication rule:

```text
∂(x2*w2)/∂x2 = w2
∂(x2*w2)/∂w2 = x2
```

Forward values:

```text
x2 = 0
w2 = 1
```

Therefore:

```text
x2.grad = w2.data * x2w2.grad
        = 1 * 0.5
        = 0.5

w2.grad = x2.data * x2w2.grad
        = 0 * 0.5
        = 0
```

So:

```python
x2.grad = 0.5
w2.grad = 0.0
```

Important intuition:

```text
w2 has zero gradient because x2 is zero
```

If the input is zero, changing the weight connected to it has no effect on the neuron output at that moment.

Since:

```text
x2*w2 = 0*w2 = 0
```

wiggling `w2` changes nothing.

# 37. Backprop through `x1w1 = x1 * w1`

We have:

```text
x1w1 = x1 * w1
```

Incoming gradient:

```text
x1w1.grad = 0.5
```

Multiplication rule:

```text
∂(x1*w1)/∂x1 = w1
∂(x1*w1)/∂w1 = x1
```

Forward values:

```text
x1 = 2
w1 = -3
```

Therefore:

```text
x1.grad = w1.data * x1w1.grad
        = -3 * 0.5
        = -1.5

w1.grad = x1.data * x1w1.grad
        = 2 * 0.5
        = 1.0
```

So:

```python
x1.grad = -1.5
w1.grad = 1.0
```

Interpretation:

```text
increasing x1 decreases o
because w1 is negative

increasing w1 increases o
because x1 is positive
```

# 38. Full neuron gradients

The neuron:

```text
o = tanh(x1*w1 + x2*w2 + b)
```

with:

```text
x1 = 2
x2 = 0
w1 = -3
w2 = 1
b ≈ 6.88137
```

has forward output:

```text
o ≈ 0.7071
```

Manual backprop gives:

```text
o.grad = 1.0
n.grad = 0.5
b.grad = 0.5
x1w1x2w2.grad = 0.5
x1w1.grad = 0.5
x2w2.grad = 0.5
x1.grad = -1.5
w1.grad = 1.0
x2.grad = 0.5
w2.grad = 0.0
```

Most important for training:

```text
w1.grad = 1.0
w2.grad = 0.0
b.grad = 0.5
```

because weights and biases are parameters we can modify.

The input gradients exist too:

```text
x1.grad = -1.5
x2.grad = 0.5
```

but in ordinary supervised learning, inputs are data and are not updated.

# 39. What this means for training

If we wanted to increase this neuron’s output `o`, the gradients say:

```text
increase w1, because w1.grad = 1.0
do not change w2 much, because w2.grad = 0.0
increase b, because b.grad = 0.5
```

The gradient is local. It tells us what happens for tiny nudges around the current values.

This is the same kind of information used when training an entire neural network.

# 40. Problem: manual backprop is ridiculous

So far, every gradient was assigned by hand:

```python
o.grad = 1.0
n.grad = 0.5
b.grad = 0.5
x1w1x2w2.grad = 0.5
...
```

This works for a tiny graph, but not for real neural networks.

The next goal is to automate it.

The insight from the manual examples:

```text
each operation knows how to propagate its output gradient to its input gradients
```

For addition:

```text
out = a + b

a.grad += out.grad
b.grad += out.grad
```

For multiplication:

```text
out = a * b

a.grad += b.data * out.grad
b.grad += a.data * out.grad
```

For tanh:

```text
out = tanh(a)

a.grad += (1 - out.data²) * out.grad
```

So the next step is to store, inside each `Value`, a small function that knows how to do the local backward pass for that operation.
