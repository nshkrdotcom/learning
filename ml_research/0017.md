# 171. The central abstraction: `Value`

Everything depends on `Value`.

A `Value` is a scalar with extra training-related state:

```python id="b0qgur"
class Value:
    def __init__(self, data, _children=(), _op=''):
        self.data = data
        self.grad = 0.0
        self._prev = set(_children)
        self._op = _op
        self._backward = lambda: None
```

Meaning:

```text id="nx151w"
data:
    forward-pass number

grad:
    derivative of final output with respect to this Value

_prev:
    children / input Values that created this Value

_op:
    operation that created this Value

_backward:
    local function for propagating gradients to children
```

This object is enough to build a tiny autograd system.

---

# 172. Forward pass

The forward pass evaluates expressions.

Example:

```python id="7y23sb"
a = Value(2.0)
b = Value(-3.0)
c = Value(10.0)

d = a * b + c
```

Forward values:

```text id="m2zqbh"
a.data = 2
b.data = -3
a*b = -6
d.data = 4
```

But during the forward pass, the graph is also built.

For:

```python id="fvdnb1"
d = a * b + c
```

micrograd records:

```text id="cs0u60"
a and b produced intermediate e by *
e and c produced d by +
```

So the forward pass has two outputs:

```text id="nfp5z9"
1. numeric result
2. computation graph
```

---

# 173. Backward pass

The backward pass computes gradients.

Given a final scalar output, usually `loss`, calling:

```python id="nis3nw"
loss.backward()
```

computes:

```text id="nsioiv"
∂loss/∂every Value in the graph
```

The algorithm:

```text id="f8qogz"
1. topologically sort the graph
2. set loss.grad = 1
3. visit nodes in reverse topological order
4. call each node’s _backward()
5. accumulate gradients into children
```

This is reverse-mode automatic differentiation.

---

# 174. Why topological order is necessary

For backprop to be correct, a node must receive all gradient contributions from later nodes before it propagates backward.

If a node feeds into multiple later branches, those branches must all contribute first.

So we need an order where:

```text id="qn9f6k"
output is processed first
then its dependencies
then their dependencies
...
```

Topological sort gives this.

Forward topological order:

```text id="ru430a"
inputs → output
```

Backward order:

```text id="3xu297"
output → inputs
```

---

# 175. Local backward rules

Each operation knows only its local derivative.

Addition:

```python id="g9hkzu"
out = a + b
```

Local derivatives:

```text id="tuzypi"
∂out/∂a = 1
∂out/∂b = 1
```

Backward:

```python id="mqejb9"
a.grad += out.grad
b.grad += out.grad
```

Multiplication:

```python id="fiidlu"
out = a * b
```

Local derivatives:

```text id="8s1kt4"
∂out/∂a = b
∂out/∂b = a
```

Backward:

```python id="ec2n0f"
a.grad += b.data * out.grad
b.grad += a.data * out.grad
```

Power:

```python id="643ayx"
out = a ** n
```

Local derivative:

```text id="1debtf"
∂out/∂a = n * a^(n-1)
```

Backward:

```python id="mvwldg"
a.grad += n * a.data**(n-1) * out.grad
```

Exponential:

```python id="vhgb8c"
out = exp(a)
```

Local derivative:

```text id="qwto2x"
∂out/∂a = exp(a) = out.data
```

Backward:

```python id="nh1q2f"
a.grad += out.data * out.grad
```

Tanh:

```python id="xyp3qd"
out = tanh(a)
```

Local derivative:

```text id="642cje"
∂out/∂a = 1 - tanh(a)^2 = 1 - out.data^2
```

Backward:

```python id="cifila"
a.grad += (1 - out.data**2) * out.grad
```

ReLU:

```python id="o60b1o"
out = relu(a)
```

Local derivative:

```text id="82zxy7"
1 if a > 0
0 if a <= 0
```

Backward:

```python id="h4aolg"
a.grad += (out.data > 0) * out.grad
```

# 176. Chain rule as the whole engine

The engine’s mathematical foundation is only the chain rule.

If:

```text id="u9oyyh"
z depends on y
y depends on x
```

then:

```text id="pdbe66"
dz/dx = dz/dy * dy/dx
```

In micrograd terms:

```text id="qy5hwl"
input.grad += local_derivative * output.grad
```

Every `_backward()` method is just this pattern.

Backpropagation is:

```text id="ruydv6"
chain rule applied recursively backward through the graph
```

---

# 177. Why gradients use `+=`

Gradients accumulate because a value can influence the final output through multiple paths.

Example:

```python id="h4n1eu"
a = Value(3.0)
b = a + a
```

Mathematically:

```text id="dj5kkh"
b = 2a
db/da = 2
```

The two branches each contribute `1`.

So:

```python id="9avip2"
a.grad += 1
a.grad += 1
```

gives:

```text id="lsc6ay"
a.grad = 2
```

If we used assignment:

```python id="z0mnxs"
a.grad = 1
a.grad = 1
```

the second write would overwrite the first.

Therefore local backward functions must accumulate.

---

# 178. Why gradients must be reset between optimization steps

Inside one backward pass:

```text id="9dz1ik"
gradient accumulation is correct
```

Across training iterations:

```text id="qj94hs"
old gradients are stale and must be cleared
```

Correct training pattern:

```python id="jpp731"
# forward
ypred = [model(x) for x in xs]
loss = sum((yout - ygt)**2 for ygt, yout in zip(ys, ypred))

# zero grad
for p in model.parameters():
    p.grad = 0.0

# backward
loss.backward()

# update
for p in model.parameters():
    p.data += -lr * p.grad
```

Without zeroing:

```text id="c8w6kt"
new gradients get added to old gradients
```

This causes incorrect updates.

# 179. The neural network abstraction

A neural network is a particular mathematical expression made of repeated neuron computations.

A neuron computes:

```text id="dod04v"
activation = w · x + b
output = nonlinearity(activation)
```

Expanded:

```text id="zypcqj"
output = tanh(w1*x1 + w2*x2 + ... + wn*xn + b)
```

or with ReLU:

```text id="sbxp6h"
output = relu(w1*x1 + w2*x2 + ... + wn*xn + b)
```

A layer is:

```text id="u2cr94"
many neurons evaluated on the same input
```

An MLP is:

```text id="tebtnh"
layers composed sequentially
```

So:

```text id="g5vn6e"
input
→ layer
→ layer
→ output
```

# 182. Loss function

The lecture uses squared error:

```python id="6707f2"
loss = sum((yout - ygt)**2 for ygt, yout in zip(ys, ypred))
```

This produces one scalar `Value`.

Low loss means:

```text id="85bpzo"
predictions close to targets
```

High loss means:

```text id="wycgpr"
predictions far from targets
```

The loss is differentiable, so micrograd can compute gradients of the loss with respect to every parameter.

# 183. Gradient descent update

Once we call:

```python id="forvdi"
loss.backward()
```

each parameter has:

```text id="8fcwdd"
p.grad = ∂loss/∂p
```

The gradient points in the direction that increases loss.

To decrease loss:

```python id="4rla7t"
p.data += -learning_rate * p.grad
```

Equivalent:

```python id="a8qzsu"
p.data -= learning_rate * p.grad
```

This is gradient descent.

---

# 184. Complete training loop

The final training loop:

```python id="r3io6t"
for k in range(num_steps):

    # forward pass
    ypred = [n(x) for x in xs]
    loss = sum((yout - ygt)**2 for ygt, yout in zip(ys, ypred))

    # backward pass
    for p in n.parameters():
        p.grad = 0.0
    loss.backward()

    # update
    for p in n.parameters():
        p.data += -learning_rate * p.grad

    print(k, loss.data)
```

This is the complete core of neural network training.

Everything else in modern systems is mostly scaling, efficiency, or improved optimization.

# 186. What “automatic differentiation” means here

Automatic differentiation is not symbolic differentiation.

It does not produce a giant algebraic derivative expression.

It is also not merely numerical finite differences.

It computes exact derivatives up to floating-point arithmetic by applying local derivative rules through the actual computation graph.

Comparison:

```text id="9e2k5b"
symbolic differentiation:
    manipulates formulas

finite differences:
    approximate slope by evaluating f(x+h)

automatic differentiation:
    records actual computation and applies chain rule exactly
```

Micrograd is reverse-mode automatic differentiation.

Reverse-mode is efficient when:

```text id="yprmyv"
many inputs
one scalar output
```

That is exactly the neural-network training case:

```text id="u84g96"
many parameters
one loss
```

---

# 187. Why neural-network training uses reverse-mode autodiff

A neural network may have millions or billions of parameters.

We need:

```text id="aezvrv"
∂loss/∂every parameter
```

That is many derivatives of one scalar output.

Reverse-mode autodiff computes all of them efficiently in one backward pass.

That is why backpropagation is the core algorithm for deep learning.

# 190. Final mental model

A neural network is a big differentiable function:

```text id="qqqfgc"
prediction = f(data, parameters)
```

A loss function turns prediction quality into one scalar:

```text id="v6kj8o"
loss = L(prediction, target)
```

Backprop computes:

```text id="zqclzj"
∂loss/∂parameters
```

Gradient descent updates:

```text id="aqmrwa"
parameters ← parameters - learning_rate * gradient
```

Repeat many times:

```text id="utxvbu"
predictions improve
loss decreases
network behavior changes toward desired behavior
```

That is the core of training.
