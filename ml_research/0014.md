```python
xs = [
```


```text
3 input dimensions
```

# 97. Getting initial predictions

Before training, the network has random parameters.

Forward pass over all examples:

```python
ypred = [n(x) for x in xs]
```

Each `n(x)` is a scalar `Value`.

The predictions are initially random-ish because the weights are random.

Example shape of results:

```text
ypred = [
    Value(data=...),
    Value(data=...),
    Value(data=...),
    Value(data=...),
]
```

The desired outputs are:

```text
[1.0, -1.0, -1.0, 1.0]
```

The predictions are usually not close enough yet.

Training means modifying the parameters so that:

```text
ypred[0] moves toward  1
ypred[1] moves toward -1
ypred[2] moves toward -1
ypred[3] moves toward  1
```

# 98. Need a single scalar objective

To train by gradient descent, we need one scalar number that measures how bad the network is.

This scalar is the **loss**.

Requirements:

```text
loss should be low when predictions are good
loss should be high when predictions are bad
loss should be differentiable with respect to network parameters
```

Then we can compute:

```text
∂loss/∂parameter
```

for every weight and bias.

---

# 99. Mean squared error loss

Karpathy uses a sum of squared errors.

For each prediction-target pair:

```text
error = yout - ygt
loss contribution = error²
```

In code:

```python
losses = [(yout - ygt)**2 for ygt, yout in zip(ys, ypred)]
loss = sum(losses)
```

Or directly:

```python
loss = sum((yout - ygt)**2 for ygt, yout in zip(ys, ypred))
```

Meaning:

```text
for every training example:
    compare prediction to target
    square the difference
sum all squared differences
```

The loss is zero only if every prediction exactly equals its target.

---

# 100. Why square the error?

For one example:

```text
prediction = yout
target = ygt
error = yout - ygt
```

If prediction is too high:

```text
error > 0
```

If prediction is too low:

```text
error < 0
```

We do not want positive and negative errors to cancel.

Squaring does two things:

```text
1. makes all errors nonnegative
2. penalizes larger errors more strongly
```

So:

```text
small error → small loss contribution
large error → large loss contribution
perfect prediction → zero contribution
```

Alternative would be absolute value, but squared error is smooth and easy to differentiate.

---

# 101. Loss as part of the computation graph

Each prediction `yout` is a `Value`.

Each target `ygt` is a Python float.

When computing:

```python
(yout - ygt)**2
```

micrograd builds graph nodes for:

```text
subtract target
power by 2
```

Then `sum(...)` builds addition nodes combining all four loss terms.

So the loss is itself a `Value`.

The graph now contains:

```text
four forward passes through the MLP
four squared-error branches
one final summed loss node
```

This is much larger than a graph for a single prediction.

---

# 102. Backprop through the loss

Once the loss is built:

```python
loss.backward()
```

This backpropagates through:

```text
loss sum
squared error terms
four prediction graphs
all MLP layers
all neurons
all weights and biases
```

Every parameter now has a `.grad`.

Example access pattern:

```python
n.layers[0].neurons[0].w[0].grad
```

This means:

```text
gradient of loss with respect to:
first layer
first neuron
first weight
```

If that gradient is negative:

```text
increasing that weight slightly decreases the loss
```

If that gradient is positive:

```text
increasing that weight slightly increases the loss
```

Gradient sign tells the local direction of influence.

Gradient magnitude tells local sensitivity.

# 104. Gradients exist for inputs too

Because the whole graph includes input values, micrograd can compute gradients with respect to input data as well.

But in ordinary supervised learning:

```text
inputs are fixed
targets are fixed
parameters are adjustable
```

So although gradients may exist for inputs, we usually ignore them.

The important gradients are for:

```text
weights
biases
```

Those are the parameters we update.

# 105. Need a way to collect all parameters

The MLP has many parameters.

For:

```python
n = MLP(3, [4, 4, 1])
```

Number of parameters:

Layer 1:

```text
4 neurons
each has 3 weights + 1 bias
4 * 4 = 16
```

Layer 2:

```text
4 neurons
each has 4 weights + 1 bias
4 * 5 = 20
```

Layer 3:

```text
1 neuron
has 4 weights + 1 bias
1 * 5 = 5
```

Total:

```text
16 + 20 + 5 = 41 parameters
```

It is inconvenient to update them one by one.

So each module gets a `.parameters()` method.

# 106. `Neuron.parameters()`

A neuron’s parameters are:

```text
all its weights
its bias
```

Implementation:

```python
class Neuron:
    ...

    def parameters(self):
        return self.w + [self.b]
```

If a neuron has three inputs:

```text
self.w = [w1, w2, w3]
self.b = b
```

then:

```python
self.w + [self.b]
```

returns:

```text
[w1, w2, w3, b]
```

All are `Value` objects.

---

# 107. `Layer.parameters()`

A layer contains multiple neurons.

Each neuron has its own parameters.

Straightforward version:

```python
class Layer:
    ...

    def parameters(self):
        params = []
        for neuron in self.neurons:
            params.extend(neuron.parameters())
        return params
```

Karpathy then compresses this into a nested list comprehension:

```python
def parameters(self):
    return [p for neuron in self.neurons for p in neuron.parameters()]
```

Meaning:

```text
for every neuron in the layer:
    for every parameter in that neuron:
        collect parameter
```

---

# 108. `MLP.parameters()`

An MLP contains multiple layers.

Each layer has parameters.

Implementation:

```python
class MLP:
    ...

    def parameters(self):
        return [p for layer in self.layers for p in layer.parameters()]
```

Now:

```python
n.parameters()
```

returns a flat list of all parameter `Value` objects in the entire network.

For the chosen architecture:

```python
len(n.parameters())
```

returns:

```text
41
```

---

# 109. Why this mirrors PyTorch

PyTorch modules have:

```python
model.parameters()
```

which returns all trainable tensors.

Micrograd’s API mirrors that idea:

```python
n.parameters()
```

returns all trainable scalar `Value` parameters.

So the training loop can be generic:

```python
for p in n.parameters():
    p.data += ...
```

It does not need to know the network’s internal structure.

# 110. First manual gradient descent update

After computing:

```python
loss.backward()
```

each parameter has:

```text
p.data = current parameter value
p.grad = derivative of loss with respect to parameter
```

Gradient descent updates parameters to reduce loss.

The gradient points in the direction of increasing loss.

So to reduce loss, move in the opposite direction:

```python
p.data += -learning_rate * p.grad
```

or:

```python
p.data -= learning_rate * p.grad
```

Karpathy uses a step size such as:

```python
0.01
```

or experiments with larger values like:

```python
0.1
```

---

# 111. Sign intuition for gradient descent

Suppose:

```text
p.grad = negative
```

This means:

```text
increasing p decreases loss
```

because the derivative of loss with respect to `p` is negative.

Update:

```python
p.data += -lr * p.grad
```

If `p.grad` is negative:

```text
-lr * p.grad is positive
```

so `p.data` increases.

That is correct.

Suppose:

```text
p.grad = positive
```

This means:

```text
increasing p increases loss
```

So to reduce loss, `p` should decrease.

Update:

```text
-lr * positive = negative
```

so `p.data` decreases.

That is also correct.

# 112. Recompute the loss after one update

After updating all parameters:

```python
for p in n.parameters():
    p.data += -0.01 * p.grad
```

recompute predictions:

```python
ypred = [n(x) for x in xs]
```

recompute loss:

```python
loss = sum((yout - ygt)**2 for ygt, yout in zip(ys, ypred))
```

The new loss should be lower.

This confirms that the gradient direction was useful.

The process is:

```text
forward pass
backward pass
parameter update
forward pass again
```

# 113. Iterating gradient descent

Training repeats the same cycle:

```text
1. forward pass: compute predictions
2. compute loss
3. backward pass: compute gradients
4. update parameters
5. repeat
```

Each iteration should generally reduce the loss if the learning rate is reasonable.

Karpathy manually repeats the procedure a few times and observes the loss decreasing.

The predictions move toward their targets:

```text
target  1 → prediction becomes more positive
target -1 → prediction becomes more negative
```

Eventually, for this tiny dataset, predictions become close to:

```text
[1, -1, -1, 1]
```

---

# 114. Learning rate

The learning rate controls step size:

```python
learning_rate = 0.01
```

Update:

```python
p.data += -learning_rate * p.grad
```

If learning rate is too small:

```text
training is stable but slow
```

If learning rate is too large:

```text
training may overshoot
loss may jump or explode
```

Karpathy demonstrates that large updates can briefly destabilize training.

The gradient only describes the local shape of the loss.

If you step too far, you may move into a region where that local approximation is no longer reliable.

---

# 115. Local nature of the gradient

The gradient answers:

```text
if I change this parameter by a tiny amount,
what happens to the loss?
```

It does not guarantee that a large change will help.

So gradient descent assumes:

```text
small enough steps
```

The update is based on local linear approximation.

This is why learning rate tuning matters.
