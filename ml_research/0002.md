## 1. Implementing the Automatic `_backward()` Step (1:09:05 – 1:17:35)
To eliminate manual backpropagation, each operation inside the `Value` class must store its own local chain-rule step during the forward pass. This is achieved by defining a private `self._backward` callable (closure) that updates the gradients of its inputs.

*   **Initialization:** By default, leaf nodes have no parents or operations, so their `_backward` is initialized to an empty function: `self._backward = lambda: None`.
*   **Operational Closures:**
    Inside each operational method, an internal `_backward` function captures references to `self`, `other` (if binary), and the output node `out`.

```python
import math

class Value:
    def __init__(self, data, _children=(), _op='', label=''):
        self.data = data
        self.grad = 0.0
        self._prev = set(_children)
        self._op = _op
        self.label = label
        self._backward = lambda: None  # Default: do nothing

    def __repr__(self):
        return f"Value(data={self.data}, grad={self.grad})"

    def __add__(self, other):
        out = Value(self.data + other.data, (self, other), '+')
        
        def _backward():
            # Sum rule: local gradients are 1.0. Chain rule multiplies by out.grad.
            self.grad = 1.0 * out.grad
            other.grad = 1.0 * out.grad
        out._backward = _backward
        
        return out

    def __mul__(self, other):
        out = Value(self.data * other.data, (self, other), '*')
        
        def _backward():
            # Product rule: local derivatives are the opposite values.
            self.grad = other.data * out.grad
            other.grad = self.data * out.grad
        out._backward = _backward
        
        return out

    def tanh(self):
        x = self.data
        t = (math.exp(2*x) - 1) / (math.exp(2*x) + 1)
        out = Value(t, (self,), 'tanh')
        
        def _backward():
            # Derivative of tanh(x) is 1 - tanh^2(x)
            self.grad = (1 - t**2) * out.grad
        out._backward = _backward
        
        return out
```

## 2. Implementing a Topological Sort (1:17:36 – 1:22:31)
*   **The Dependency Ordering Problem:** We cannot evaluate `_backward()` on a node until all nodes that depend on it have already computed their gradients. Backpropagation must proceed in reverse topological order.
*   **Topological Sort on a DAG:** An ordering of nodes in a directed acyclic graph such that for every directed edge $U \to V$, node $U$ comes before $V$ in the ordering. For the backward pass, we reverse this list so that the root node is evaluated first, propagating gradients back to the leaves.
*   **Algorithm (DFS Graph Traversal):**
    ```python
    def backward(self):
        # Build topological order
        topo = []
        visited = set()
        
        def build_topo(v):
            if v not in visited:
                visited.add(v)
                for child in v._prev:
                    build_topo(child)
                topo.append(v)
                
        build_topo(self)
        
        # Base case: the output node's gradient with respect to itself is 1.0
        self.grad = 1.0
        
        # Go backwards through the topological list and call _backward()
        for node in reversed(topo):
            node._backward()
    ```

## 3. Identifying and Fixing the Gradient Overwriting Bug (1:22:32 – 1:27:04)
*   **The Bug Scenario:** If a variable is reused multiple times in an expression, its gradient is overwritten rather than accumulated.
    *   *Example 1:* Let $b = a + a$. Analytically, $\frac{\partial b}{\partial a} = 2$.
    *   *Under the code in Section 1:*
        $$\text{\texttt{out.\_backward()}} \implies \text{\texttt{self.grad}} = 1.0 \cdot \text{\texttt{out.grad}}$$
        $$\text{\texttt{other.grad}} = 1.0 \cdot \text{\texttt{out.grad}}$$
        Because `self` and `other` point to the same object `a`, the second assignment overrides the first. Thus `a.grad` remains $1.0$ instead of $2.0$.
    *   *Example 2:* A variable branching to two different downstream nodes. The gradient signals coming back from each path will overwrite each other depending on the order of execution in the topological list.
*   **Mathematical Justification (Multivariate Chain Rule):**
    If a variable $x$ flows to multiple variables $y_1, y_2, \dots, y_n$, the total derivative of the loss $L$ with respect to $x$ is the sum of the partial derivatives along all branches:
    $$\frac{\partial L}{\partial x} = \sum_{i} \frac{\partial L}{\partial y_i} \frac{\partial y_i}{\partial x}$$
*   **The Fix:** Accumulate gradients using `+=` instead of direct assignment `=`. All gradients must start at $0.0$ so they can sum up properly.

```python
# Updated definitions with gradient accumulation (+=)
def __add__(self, other):
    out = Value(self.data + other.data, (self, other), '+')
    def _backward():
        self.grad += 1.0 * out.grad
        other.grad += 1.0 * out.grad
    out._backward = _backward
    return out

def __mul__(self, other):
    out = Value(self.data * other.data, (self, other), '*')
    def _backward():
        self.grad += other.data * out.grad
        other.grad += self.data * out.grad
    out._backward = _backward
    return out
```

## 4. Extending the Scalar Engine with More Operations (1:27:05 – 1:39:32)
To build more complex expressions, the engine must support non-`Value` inputs (e.g., standard integers and floats), division, subtraction, powers, and exponential functions.

### A. Fallbacks for Constant Inputs (`isinstance` and `__rmul__`/`__radd__`)
If we evaluate `a + 1` or `a * 2` where `a` is a `Value` and the other operand is a Python `float` or `int`, we must wrap the constant operand in a `Value` automatically:
```python
def __add__(self, other):
    other = other if isinstance(other, Value) else Value(other)
    out = Value(self.data + other.data, (self, other), '+')
    # ... rest of __add__ code ...
```
If we evaluate `2 * a` or `1 + a`, Python looks for a left-side operator on `int`, which does not support `Value`. It then falls back to the right-side operators (`__radd__` and `__rmul__`):
```python
def __radd__(self, other): # fallback for other + self
    return self + other

def __rmul__(self, other): # fallback for other * self
    return self * other
```

### B. The Power Rule (`__pow__`)
Allows raising a `Value` to a constant integer or float power, $x^k$.
*   **Analytical Derivative:** $\frac{d}{dx}(x^k) = k \cdot x^{k-1}$.
*   **Implementation:**
    ```python
    def __pow__(self, other):
        assert isinstance(other, (int, float)), "Only supporting int/float powers for now"
        out = Value(self.data ** other, (self,), f'**{other}')
        
        def _backward():
            self.grad += (other * (self.data ** (other - 1))) * out.grad
        out._backward = _backward
        
        return out
    ```

### C. Division and Subtraction via Primitives
Instead of defining complex custom backward logic for division ($\frac{a}{b}$) and subtraction ($a - b$), we can represent them using addition, multiplication, negation, and powers:
*   **Negation:** $-a \equiv a \cdot -1$
    ```python
    def __neg__(self):
        return self * -1
    ```
*   **Subtraction:** $a - b \equiv a + (-b)$
    ```python
    def __sub__(self, other):
        return self + (-other)

    def __rsub__(self, other):
        return other + (-self)
    ```
*   **Division:** $\frac{a}{b} \equiv a \cdot b^{-1}$
    ```python
    def __truediv__(self, other):
        return self * (other**-1)

    def __rtruediv__(self, other):
        return other * (self**-1)
    ```

### D. Exponential Function (`exp`)
Required to construct sigmoid or hyperbolic tangent activation functions manually.
*   **Analytical Derivative:** $\frac{d}{dx}(e^x) = e^x$.
*   **Implementation:**
    ```python
    def exp(self):
        x = self.data
        out = Value(math.exp(x), (self,), 'exp')
        
        def _backward():
            self.grad += out.data * out.grad  # local derivative is out.data (e^x)
        out._backward = _backward
        
        return out
    ```

## E. Deconstructing $\tanh$ using Exponentials, Division, and Subtraction
Using the newly implemented math operators, we can break down a single composite activation node like $\tanh(n)$ into its fundamental components:
$$o = \tanh(n) = \frac{e^{2n} - 1}{e^{2n} + 1}$$

```python
# Setup equivalent to the single-neuron example
n = Value(0.8813735870195432)

# Deconstructed tanh expression
# e2n = exp(2 * n)
e2n = (n * 2).exp()
# o = (e2n - 1) / (e2n + 1)
o = (e2n - 1) / (e2n + 1)

o.backward()
print(f"n.grad: {n.grad}")  # Outputs 0.5 (identical to composite tanh)
```
*   **Comparison of Composite vs. Deconstructed Graphs:**
    *   *Composite:* Representing $\tanh$ as a single node simplifies the visual expression graph. It requires fewer intermediate operations and is more computationally efficient because it uses the analytical shortcut $1 - \tanh^2(x)$.
    *   *Deconstructed:* Breaking down $\tanh$ into exponentials, division, and subtraction creates a much deeper, more complex DAG. However, because each local backward operation is correct, the accumulation of gradients through the chain rule yields the exact same numerical result.
