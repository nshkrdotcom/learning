Karpathy micrograd outline — Part 1 of 8

Scope: intro → derivatives → first Value object → expression graph visualization → setup for backprop

Approx. video range: 0:00–32:10

1. Goal of the lecture

Karpathy’s purpose is to show neural network training under the hood.

The end state of the lecture:

blank notebook
→ build a tiny autograd engine
→ build a tiny neural net library on top
→ train a neural network

The core object of study is micrograd.

Micrograd is:

a tiny automatic differentiation engine

More specifically:

micrograd = scalar-valued autograd engine
autograd = automatic gradient
automatic gradient = automatic backpropagation

Backpropagation is the algorithm that efficiently computes:

gradient of loss with respect to parameters

For a neural network, this means:

how does each weight affect the loss?

Once you know that, you can update weights to make the loss smaller.

2. What micrograd does

Micrograd lets you build mathematical expressions out of scalar values.

Example structure:

a = Value(-4.0)
b = Value(2.0)

c = a + b
d = a * b + b**3
...
g = ...

The actual expression in the README is intentionally meaningless. Its purpose is to show that micrograd supports operations like:

addition
multiplication
power
negation
division
ReLU-like squashing
composition of many operations

The important point is not the specific expression.

The important point is that micrograd builds a computation graph.

For example:

c = a + b

means:

c is a Value
c.data stores the numeric result
c knows it was created by +
c points back to a and b

So each Value stores not only its number, but also its local history.

3. Forward pass vs backward pass

A forward pass evaluates the expression numerically.

Example:

g.data

returns the scalar output value.

But the important part is:

g.backward()

This runs backpropagation starting from g.

Backprop walks backward through the graph and computes derivatives such as:

dg/da
dg/db
dg/dc
...

These are stored as:

a.grad
b.grad
c.grad

Interpretation:

a.grad = how much g changes if a is nudged slightly upward
b.grad = how much g changes if b is nudged slightly upward

If:

a.grad = 138

then a tiny positive change to a should increase g at slope roughly 138.

If:

b.grad = 645

then b has an even stronger positive local influence on g.

4. Neural networks are just mathematical expressions

Karpathy emphasizes that neural networks are not a special case for backprop.

They are just mathematical expressions.

A neural network takes as input:

input data
weights / parameters

and produces:

predictions

Then the predictions feed into a loss function:

loss = measure of how bad the predictions are

So the total system is still one mathematical expression:

data, weights → neural net → predictions → loss

Backpropagation does not care that this expression is a “neural network.”

It only sees:

operations connected in a graph

The same machinery works for arbitrary scalar expressions.

5. Why micrograd uses scalars, not tensors

Modern libraries like PyTorch operate on tensors.

A tensor is basically:

an n-dimensional array of scalars

Micrograd instead operates on individual scalar values.

This is inefficient but pedagogically clean.

The point:

scalar micrograd = understand the math
tensor PyTorch = same math, packed for speed

Tensors do not change the underlying calculus. They make computation efficient by batching many scalar operations together and exploiting parallelism.

Karpathy’s claim:

micrograd contains the conceptual essence of neural network training;
everything else is efficiency.

He points out that micrograd’s core engine is about 100 lines, and the neural-net library on top is similarly tiny.

6. Derivative intuition: single-input function

Karpathy starts with a scalar function:

f(x) = 3x² - 4x + 5

It maps:

one scalar input x
→ one scalar output y

For example:

f(3.0) = 20

He plots the function over a range of x-values, showing a parabola.

The goal is to understand the derivative intuitively.

The derivative at a point means:

if I nudge x slightly upward,
how does f(x) respond?

Formal definition:

f'(x) = lim h→0 [f(x + h) - f(x)] / h

In code, approximate with small h:

h = 0.001
x = 3.0
(f(x + h) - f(x)) / h

At x = 3, the derivative is about:

14

Meaning:

near x = 3, increasing x slightly increases f(x)
with slope about 14

Analytically:

f(x) = 3x² - 4x + 5
f'(x) = 6x - 4
f'(3) = 18 - 4 = 14

At x = -3, the slope is negative:

f'(-3) = -22

Interpretation:

near x = -3, increasing x makes f(x) decrease

At the bottom of the parabola, the derivative is zero.

For this function, that point is:

x = 2/3

At that point:

a tiny nudge in x barely changes f(x)

7. Derivative intuition: multiple-input function

Next he uses a function with three scalar inputs:

d = a * b + c

with values:

a = 2
b = -3
c = 10

Then:

d = 2 * -3 + 10 = 4

Now there are multiple derivatives:

∂d/∂a
∂d/∂b
∂d/∂c

Each asks:

if I nudge this one input slightly upward,
holding the others fixed,
how does d respond?

Using the finite-difference method:

h = 0.0001

a += h
d2 = a * b + c

slope = (d2 - d1) / h

For a:

∂d/∂a = b = -3

Intuition:

a increases
but b is negative
so a*b becomes more negative
so d decreases

For b:

∂d/∂b = a = 2

Intuition:

b increases
a is positive
so a*b increases
so d increases

For c:

∂d/∂c = 1

Intuition:

d includes +c directly
so increasing c by h increases d by h

This establishes the basic meaning of gradients:

gradient = local sensitivity of output to inputs

8. Begin building micrograd: the Value object

Neural networks are huge mathematical expressions, so we need a data structure to represent expression graphs.

Karpathy introduces:

class Value:
    def __init__(self, data):
        self.data = data

A Value wraps a scalar number.

Example:

a = Value(2.0)

To make printing readable, he implements:

def __repr__(self):
    return f"Value(data={self.data})"

So instead of an ugly Python object representation, we get:

Value(data=2.0)

9. Adding operations: __add__

Python does not automatically know how to add two Value objects.

So:

a + b

initially fails.

Karpathy defines:

def __add__(self, other):
    out = Value(self.data + other.data)
    return out

Now:

Value(2.0) + Value(-3.0)

produces:

Value(data=-1.0)

Important distinction:

Value + Value

is implemented by extracting their raw Python numbers:

self.data + other.data

then wrapping the result in a new Value.

10. Adding multiplication: __mul__

Similarly:

def __mul__(self, other):
    out = Value(self.data * other.data)
    return out

Now expressions like this work:

a = Value(2.0)
b = Value(-3.0)
c = Value(10.0)

d = a * b + c

Forward result:

2 * -3 + 10 = 4

So:

d = Value(data=4.0)

At this point, micrograd can do forward computation but does not yet know graph structure.

11. Storing graph connectivity

To backpropagate, each output must know which previous values created it.

Karpathy modifies Value:

class Value:
    def __init__(self, data, _children=()):
        self.data = data
        self._prev = set(_children)

Now when creating a result from addition:

out = Value(self.data + other.data, (self, other))

and multiplication:

out = Value(self.data * other.data, (self, other))

So if:

d = a * b + c

then d can point backward to the nodes that produced it.

For example:

d._prev = {e, c}

where:

e = a * b

This creates the computation graph.

12. Storing the operation

Knowing the previous nodes is not enough.

We also want to know what operation produced the value.

So the constructor becomes:

def __init__(self, data, _children=(), _op=''):
    self.data = data
    self._prev = set(_children)
    self._op = _op

For addition:

out = Value(self.data + other.data, (self, other), '+')

For multiplication:

out = Value(self.data * other.data, (self, other), '*')

Now each node knows:

its scalar data
which previous nodes made it
which operation made it

That is enough to represent the mathematical expression.

13. Visualizing the expression graph

Karpathy introduces graph visualization code using Graphviz.

The visualization has two helper ideas:

nodes = all Value objects in the graph
edges = parent-child relationships between Value objects

The helper function trace(root) walks backward from the final node and collects all nodes and edges.

Then draw_dot(root) visualizes them.

Important detail:

The actual computation graph contains only Value objects.

But for visualization, he inserts fake operation nodes:

+
*
tanh
etc.

These operation nodes are not real Value objects. They make the graph easier to read.

14. Adding labels

To make diagrams readable, he adds labels:

def __init__(self, data, _children=(), _op='', label=''):
    self.data = data
    self.label = label

Then:

a.label = 'a'
b.label = 'b'
c.label = 'c'
e.label = 'e'
d.label = 'd'

Expression:

a = Value(2.0, label='a')
b = Value(-3.0, label='b')
c = Value(10.0, label='c')

e = a * b
e.label = 'e'

d = e + c
d.label = 'd'

Graph:

a ----\
      * → e ----\
b ----/          + → d
c ---------------/

Numerically:

a = 2
b = -3
e = a*b = -6
c = 10
d = e+c = 4

15. Making the graph deeper

He adds another value:

f = Value(-2.0, label='f')
L = d * f
L.label = 'L'

Now:

d = 4
f = -2
L = d*f = -8

Graph:

a ----\
      * → e ----\
b ----/          + → d ----\
c ---------------/          * → L
f --------------------------/

This is the first complete toy computation graph used for backprop.

16. What backprop will compute

The forward pass computes:

L = -8

Backprop will compute derivatives of L with respect to every node:

∂L/∂L
∂L/∂d
∂L/∂f
∂L/∂e
∂L/∂c
∂L/∂a
∂L/∂b

In neural networks, the final node is usually the loss:

L = loss

The most important derivatives are:

∂loss/∂weight

because weights are what we can change.

The data inputs also technically get gradients, but usually we do not use them for training because the data is fixed.

17. Adding the .grad field

Karpathy adds:

self.grad = 0.0

to every Value.

So each node stores:

data = forward-pass scalar value
grad = derivative of final output with respect to this node

Initially:

grad = 0

Why zero?

Because before backprop, we assume:

this node has no known influence on the output

Backprop will fill in the actual sensitivities.

The visualization is updated so each node shows:

label
data
grad

Example:

a | data 2.0000 | grad 0.0000

At this moment, all gradients are zero.

The next step is manual backpropagation through the graph.
